# Grant Fit

## Why this is a strong Frontier Pool fit

Celo Agent Preflight is infrastructure, not an end-user AI app. It helps builders and agents verify whether an ERC-8004/x402/MCP/A2A agent is actually usable on Celo.

It maps to the Frontier Pool categories:

- Agent identity and discovery: validates ERC-8004 registration and metadata.
- Agent-to-agent transaction infrastructure: checks x402-style payment readiness on Celo.
- AI-native developer tooling: ships as MCP, CLI, API, and GitHub Action.
- Verification and trust infrastructure: produces readiness reports and Celo mainnet attestations.
- Interoperability: checks MCP, A2A, x402, ERC-8004, Self Agent ID, and Celo activity together.

## Main risk

The project must avoid looking like a thin explorer or badge site. It should be positioned as a conformance/readiness layer with reproducible checks and mainnet evidence.

## Highest-scoring proof points

- Open-source repo with runnable CLI/MCP.
- Celo mainnet attestation contract deployed and verified.
- Reports for existing Celo agents.
- Readiness report hash written onchain.
- Agents/builders adding badge or report link to their GitHub/docs/metadata.
- Demo showing an agent querying Preflight before calling/paying another agent.
