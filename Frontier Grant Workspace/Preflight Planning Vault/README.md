# Celo Agent Preflight

An Obsidian-compatible planning vault for the Frontier grant application and MVP delivery.

Core thesis: **Before an agent ships, calls, or pays another agent, run preflight.**

Product model: **Preflight is the readiness engine; ReadyList is the ChainList-style public directory of scanned Celo agents powered by Preflight reports.**

## Suggested workflow

1. Keep all raw notes in `00-Inbox`.
2. Use `01-Research` for sourced claims and market context.
3. Write final form answers in `02-Grant-Application`.
4. Track product decisions in `03-Product`.
   - Start with `03-Product/ReadyList-Spec.md` for the public directory model.
5. Track implementation in `04-Technical`.
6. Track activation and partnerships in `05-Go-To-Market`.
7. Track mainnet transactions, reports, videos, and proof in `06-Evidence`.
8. Track KYC, address, logo, open-source readiness, and submission ops in `07-Operations`.

## Immediate priority

The grant application should not read like a pitch deck only. It should show proof:

- Celo mainnet deployment
- ERC-8004 registration or clear AI-infrastructure exemption reasoning
- Self Agent ID integration plan/status
- public GitHub repository
- verifiable mainnet attestation transaction
- at least 3 to 10 real agent readiness reports
- ReadyList index showing those reports as reusable public infrastructure
- demo video
- explicit activation path to first 10 builders and first 100 agents
